# Прогнозирование оттока клиентов телеком-компании

Проект посвящён прогнозированию оттока клиентов телеком-компании. В работе выполнены очистка и анализ данных, исследование дисбаланса классов, сравнение моделей машинного обучения, кросс-валидация и подбор порога классификации.

## Задача

Построить модель бинарной классификации, которая определяет, уйдёт ли клиент из компании:

- `0` --- клиент останется;
- `1` --- клиент уйдёт.

Основной метрикой выбрана **F1-score**, поскольку целевой класс представлен заметно реже основного и важно учитывать одновременно Precision и Recall.

## Данные

Использован [Iranian Churn Dataset](https://archive.ics.uci.edu/dataset/563/iranian%2Bchurn%2Bdataset) из UCI Machine Learning Repository.

Датасет содержит информацию об использовании клиентами услуг телеком-компании:

- количество звонков;
- продолжительность использования услуг;
- частоту звонков и SMS;
- срок подписки;
- тарифный план;
- возраст клиента;
- ценность клиента;
- целевой признак `Churn`.

После первичной обработки:

- исходный размер --- 3150 записей и 16 столбцов;
- удалено 300 полных дубликатов;
- после очистки осталось 2850 записей;
- признаки `FN` и `FP` не использовались при обучении;
- целевой столбец --- `Churn`.

## Этапы работы

1.  Загрузка и первичный анализ данных.
2.  Поиск и удаление полных дубликатов.
3.  Анализ распределения целевого класса.
4.  Исключение технических и расчётных признаков.
5.  Стратифицированное разделение данных на обучающую и тестовую выборки.
6.  Сравнение моделей с балансировкой классов и без неё.
7.  Пятикратная стратифицированная кросс-валидация.
8.  Подбор порога классификации по OOF-прогнозам.
9.  Проверка выбранных конфигураций на независимой тестовой выборке.
10. Анализ ROC-кривых, матриц ошибок и важности признаков.

## Модели

В проекте сравнивались:

- Logistic Regression;
- Random Forest;
- Gradient Boosting.

## Результаты

| Модель | Балансировка | Порог | CV F1 | Test F1 | Precision | Recall | ROC-AUC |
|----|----|---:|---:|---:|---:|---:|---:|
| Random Forest | Без балансировки | 0.405 | 0.863 | **0.891** | **0.907** | 0.876 | 0.985 |
| Gradient Boosting | Без балансировки | 0.375 | 0.861 | 0.876 | 0.844 | **0.910** | **0.987** |
| Logistic Regression | Без балансировки | 0.250 | 0.681 | 0.657 | 0.559 | 0.798 | 0.928 |

Лучший результат по основной метрике F1-score показал **Random Forest**:

- F1-score --- **0.891**;
- Precision --- **0.907**;
- Recall --- **0.876**;
- ROC-AUC --- **0.985**.

Gradient Boosting показал более высокие Recall и ROC-AUC, поэтому выбор итоговой модели зависит от бизнес-задачи:

- Random Forest лучше сохраняет баланс между Precision и Recall;
- Gradient Boosting обнаруживает больше клиентов, склонных к уходу, но формирует больше ложных срабатываний.

## Основные выводы

- Автоматическая балансировка классов не улучшила итоговую F1-score исследованных моделей.
- Подбор порога классификации оказался полезнее использования стандартного порога `0.5`.
- Наиболее высокое качество показали ансамблевые модели.
- Результаты моделей следует интерпретировать как прогноз риска оттока, а не как доказательство причин ухода клиентов.

## Запуск проекта

Клонировать репозиторий:

    git clone <ссылка-на-репозиторий>
    cd telecom-customer-churn

Создать виртуальное окружение:

    python -m venv .venv

Активировать его в Windows:

    .venv\Scripts\activate

Установить зависимости:

    pip install -r requirements.txt

Запустить Jupyter Notebook:

    jupyter notebook telecom_churn_analysis.ipynb

Файлы `iranian_churn.csv` и `model_worker.py` должны находиться в одной директории с ноутбуком.

- Подобранный порог зависит от распределения классов и стоимости ошибок.
